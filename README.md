## Bug Reports – QA Manual / Functional Testing

Este repositorio contiene bug reports documentados en formato Markdown, creados a partir de la ejecución de casos de prueba funcionales sobre un portal bancario ficticio, simulando un flujo de trabajo de QA Manual real.

Cada bug report describe un defecto funcional detectado durante las pruebas, con información suficiente para su correcta reproducción y análisis por parte de equipos de desarrollo.

⸻

## Contenido del repositorio

Los bug reports incluyen:
	•	Descripción clara del defecto
	•	Pasos para reproducir el error
	•	Resultado esperado vs resultado actual
	•	Severidad y prioridad
	•	Contexto funcional del problema

⸻

## Alcance de los defectos reportados

Los defectos documentados están relacionados con:
	•	Validaciones de campos obligatorios
	•	Manejo de errores y mensajes al usuario
	•	Persistencia incorrecta de datos
	•	Flujos funcionales inconsistentes

⸻

## Objetivo del repositorio

Demostrar habilidades prácticas en:
	•	Detección y documentación de defectos funcionales
	•	Redacción clara y reproducible de bug reports
	•	Análisis del impacto funcional de errores
	•	Aplicación de criterios de severidad y prioridad

Este repositorio forma parte de un portafolio QA Junior, orientado a roles de QA Manual / QA Funcional.
